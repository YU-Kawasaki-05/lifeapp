# 多数並行プロジェクト期の生産性フレームワーク

## 1. 核心的な構造認識：Exploit と Explore の二重OS

あなたが直面している問題は、組織科学で **Exploitation（活用）vs Exploration（探索）** と呼ばれるトレードオフそのものである（March, 1991）。タスクを効率的にこなす能力と、新たな可能性を発見する能力は、同じ時間・認知資源を奪い合う。

**重要な知見：** 両方を「同時に」やろうとすると両方が劣化する。研究が示すのは、**時間的に分離して交互に行う（temporal ambidexterity）** ことが個人レベルでは最も有効だということ（Gupta et al., 2006）。

→ つまり「毎日の中に両方のモードを設計する」のではなく、**週単位で明確にExploit時間とExplore時間を分ける**のが最善。

---

## 2. 週の設計：3層アーキテクチャ

### 第1層：週次リセット（週1回、60-90分）

**エビデンス：** メタ認知的モニタリング（自分の思考を俯瞰する能力）が高い人ほど、マルチプロジェクト環境で成果を出す（Kraiger & Wenzel, 1997）。定期的な振り返りがこの能力を強化する。

**実行内容：**
- 全プロジェクトの現状を1枚に書き出す（進捗・次のマイルストーン・ブロッカー）
- 「今週、最もインパクトが大きいのはどれか」を強制的に序列化する
- 「やらないことリスト」を作る（これが最重要。Drucker の「体系的廃棄」）
- 翌週の時間ブロックをざっくり配置する

**推奨タイミング：** 日曜夕方 or 月曜早朝。脳がフレッシュで、週の全体像を見渡せる。

### 第2層：日次のExploitモード（平日の大半）

ここが「早く起きて一つ一つ効率的に終わらせる」パートに該当する。

**a) ウルトラディアンリズムの活用**

人間の集中力は約90分周期で波を打つ（Kleitman のBRAC理論、Peretz Lavie の研究で支持）。

- **90分の集中ブロック → 15-20分の完全休憩** を1日3-4セット
- 1日の「深い仕事」の現実的上限は約4時間（Ericsson et al., 1993 の熟達研究）
- 残りの時間はメール・会議・軽い作業に充てる

**b) コンテキストスイッチングの最小化**

Gloria Mark の研究（2004, 2008）によると、中断後に元のタスクに完全復帰するまで平均23分かかる。複数プロジェクトを1日の中で頻繁に切り替えると、この「復帰コスト」で1-2時間が消える。

**対策：**
- **1つの90分ブロック = 1つのプロジェクト**を厳守
- 1日に扱うプロジェクトは最大2-3個に制限
- 「今日はA, B, C」「明日はD, E」のように日単位でプロジェクトをバッチング
- 類似タスク（メール返信、レビュー、事務処理）は1つのブロックにまとめる

**c) タスク開始の摩擦を下げる**

Zeigarnik効果の応用：前日の終わりに、翌朝最初にやることの「最初の一手」だけメモしておく。「○○のスライドの3ページ目のグラフを直す」レベルの具体性。これにより朝の立ち上がりが劇的に速くなる。

**d) 朝の使い方**

認知心理学のresearch on circadian rhythmsによれば、起床後2-4時間が分析的思考のピーク（Wieth & Zacks, 2011）。

- 起床後すぐ：メール・SNSを見ない（注意資源の浪費を防ぐ）
- 最初の90分ブロック：その日最も認知的に重い仕事に充てる
- 2番目の90分ブロック：次に重い仕事
- 午後：会議、コミュニケーション、軽いタスク

### 第3層：Exploreモード（週に最低1ブロック、理想は2ブロック）

**エビデンス：** Googleの「20%ルール」の背後にある原理。完全にタスク駆動の状態が続くと、認知的固着（cognitive fixation）が起き、問題解決の質が落ちる（Smith, 1995）。また、弱い紐帯（weak ties）からのセレンディピティが長期的な成果を決定づける（Granovetter, 1973）。

**実行内容（以下から選択・組合せ）：**
- 業界外の人との雑談・ランチ
- 自分の専門外の本・記事を読む
- 「もし制約がなかったら何をするか」を書く
- 小さな実験を1つ走らせる（失敗してもいいプロトタイプ）
- 散歩しながら音声メモ

**重要：** Exploreの時間は「余った時間にやる」と絶対にやらない。**カレンダーにブロックして予約する**。これは「贅沢」ではなく、将来のExploitの種を作る投資である。

## 3. メンタルと創造性の維持

### a) 認知的余白の確保

**エビデンス：** DMN（デフォルトモードネットワーク）は、ぼんやりしている時に活性化し、創造的な結合や自己参照的思考を担う（Buckner et al., 2008）。常にタスクに追われている状態はDMNの活動を抑制する。

**実践：**
- 移動中にポッドキャストやSNSを見ない時間を作る
- シャワー・散歩・単純作業中に意図的に「何もしない」
- 1日15分でも「予定のない時間」を持つ

### b) 運動

これは最もエビデンスが強い介入の一つ。週150分の中強度運動が、実行機能（計画、切り替え、抑制）を有意に向上させる（Hillman et al., 2008）。さらに、運動直後の数時間は認知パフォーマンスが上昇する。

→ 朝の仕事前に20-30分の運動を入れると、最初の90分ブロックの質が上がる。

### c) 睡眠の死守

睡眠を削って作業時間を増やすのは、研究が一貫して否定している戦略（Walker, 2017; Van Dongen et al., 2003）。6時間以下の睡眠が2週間続くと、認知機能は48時間徹夜と同等まで低下する。しかも本人は劣化に気づかない。

→ 早起きは有効だが、就寝時間を前倒しにすることとセットでなければ逆効果。7-8時間を確保する。

### d) 「完了」の感覚を意図的に作る

多数並行の最大のストレス要因は「何も終わらない感覚」（Zeigarnik効果のネガティブ面）。

- 大きなプロジェクトを小さなマイルストーンに分割し、「今日はここまで完了」を明示する
- 1日の終わりに「Done List（やったことリスト）」を3行書く
- 週次レビューで完了項目を確認する

## 4. 危険信号と調整ルール

以下の兆候が出たら、負荷を下げるか構造を見直す必要がある：

- 2日連続で最も重要なタスクに手をつけられない → 優先順位の再設定が必要
- 睡眠が6時間を切る日が3日続く → 何かを捨てる
- Exploreの時間を2週連続でスキップ → 短期最適化の罠に入っている
- 「忙しい」が口癖になっている → 構造の問題であり、努力の問題ではない

## 5. 1日のテンプレート例

| 時間帯 | 活動 | モード |
|---|---|---|
| 5:30-6:00 | 起床・軽い運動 | 準備 |
| 6:00-7:30 | 最重要タスク（深い集中） | Exploit |
| 7:30-7:50 | 休憩 | 回復 |
| 7:50-9:20 | 2番目に重要なタスク | Exploit |
| 9:20-10:00 | メール・Slack処理 | バッチ処理 |
| 10:00-11:30 | 3番目のタスク or 会議 | Exploit |
| 11:30-13:00 | 昼食・散歩（認知的余白） | 回復 |
| 13:00-14:30 | 会議 or 軽いタスク | 低負荷 |
| 14:30-15:30 | コミュニケーション・レビュー | バッチ処理 |
| 15:30-16:00 | 翌日の準備・Done List記入 | 振り返り |

※ Exploreブロック（週2回）は、上記の1つのExploitブロックを置き換える形で入れる。

## 6. 最も重要な1つのこと

全体を貫く最重要原則は **「やらないことを決める勇気」** である。

Peter Drucker の言葉を借りれば、「成果をあげる人は仕事からスタートしない。時間からスタートする」。持ち時間は有限で、増やせない。できることは、その中に何を入れ、何を入れないかを設計することだけ。

多数並行の時期こそ、「全部やる」ではなく「どれを90点にし、どれを60点で許容するか」を意識的に決める。完璧主義を手放す構造を自分に与えること。それが、タスクに埋もれず、拡張し続けるための最も確実な方法である。
