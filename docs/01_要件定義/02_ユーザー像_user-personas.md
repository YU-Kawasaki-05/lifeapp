# ユーザー像定義書
## プロジェクト名: ARDORS（アーダース）

---

### 1. ユーザー種別一覧

| ID | ユーザー種別 | 説明 | 想定人数 | 利用頻度 | スコープ |
|----|------------|------|---------|---------|---------|
| U-01 | 一般ユーザー | ARDORSを日常的に使い、自分の生産性・プロジェクト・習慣・目標を管理する人 | 多数（SaaS公開後） | 毎日 | 初期リリース |
| U-02 | システム管理者 | SaaSの運用管理を行う人（初期は開発者本人） | 1名（初期） | 必要時 | 初期リリース |
| U-03 | ゲスト（閲覧のみ） | ログインなしで一般ユーザーの公開カレンダー（空き時間）を閲覧し、予約を行う人 | 不定 | 不定 | 将来拡張 |

### 2. ペルソナ詳細

#### U-01: 一般ユーザー

- **役割**: 自分自身の生産性・プロジェクト・習慣・目標をARDORSで管理する
- **主な利用目的**: 多数のプロジェクトを着実に前進させ、長期目標に向かって成長する
- **代表的なプロフィール**:
  - **ペルソナA: 多忙な大学生**
    - 就活、学業、サークル、アルバイト、趣味、友人関係を同時並行
    - 期限が多い（レポート、ES提出、面接日程）
    - 自分でスケジュールを管理する経験が浅く、迷いやすい
    - スマホメインで使いたいが、レポート作業時はPC
  - **ペルソナB: フリーランス/個人開発者**
    - 複数のクライアント案件、自社プロダクト開発、スキルアップ、営業活動を並行
    - 自分で全てを管理する必要があり、管理コストが高い
    - 長期的なキャリアビジョンと短期の納期の間で優先順位に悩む
    - PCメインだが、外出先ではスマホ
  - **ペルソナC: 副業・起業準備中の社会人**
    - 本業と副業/起業準備を並行。限られた時間で最大の成果を出したい
    - 運動や健康管理も含めた人生全体のバランスが課題
    - 目先のタスクに追われて、本当にやりたいことに時間を使えていない
- **利用シーン**:
  - 朝: ARDORSを開いてモーニングブリーフィングを確認し、1日を開始
  - 日中: タイムブロックに沿って作業。ブロック間で1タップの振り返り
  - 夕方: デイリークローズで振り返り。まとめて音声/テキスト入力 → AI分析
  - 週末: ウィークリーレビューで方向修正と来週計画。AIがタイムボクシング生成
  - 随時: 思いついたことをブレインダンプ（音声/テキスト）
- **利用デバイス**: PC（ブラウザ）+ スマホ（ブラウザ）。両方Web版
- **ITリテラシー**: 中〜高（日常的にデジタルツールを使う層）
- **主要タスク**:
  1. AIと対話してスケジュール・タスクの提案を受け、承認する
  2. プロジェクトの進捗を確認・更新する
  3. 習慣の実行を記録する
  4. 振り返り（日次/週次/月次）を行い、AIからフィードバックを受ける
  5. ブレインダンプで思考を吐き出し、AIに構造化してもらう
  6. ダッシュボードで全体像を俯瞰する（目標〜今日のタスクの接続を確認）
  7. Google Calendarから予定をpull / タイムボクシングをpush
  8. ナレッジ・気づきをキャプチャする
- **ペインポイント**:
  - 情報が分散していて一覧性がない
  - スケジュール設計の認知負荷が高すぎる
  - 優柔不断で何から手をつけるか迷う
  - 長期目標から乖離し、成長実感がない
  - 達成感がなく、何ができたかわからない
  - 管理がめんどくさい。入力コストは最小にしたい
- **成功体験**:
  - 朝ARDORSを開くだけで今日何をすべきかわかる
  - プロジェクトがガンガン前に進んでいる実感がある
  - 週次レビューで自分の成長を客観的に把握できる
  - AIが現実的なスケジュールを組んでくれて、無理なく予定通り進む

#### U-02: システム管理者

- **役割**: ARDORSのSaaS運用を管理する（初期は開発者本人）
- **主な利用目的**: ユーザー管理、システム設定、運用状況の把握
- **利用シーン**:
  - ユーザーからの問い合わせ対応時
  - システムの異常検知・対応時
  - ユーザー数・利用状況のモニタリング
  - 将来的なプラン管理（無料/有料の機能差設定）
- **利用デバイス**: PC（ブラウザ）
- **ITリテラシー**: 高（開発者）
- **主要タスク**:
  1. ユーザーアカウントの管理（一覧、検索、停止、削除）
  2. システム設定の変更
  3. 利用状況のダッシュボード閲覧（ユーザー数、アクティブ率等）
  4. 将来: プラン管理（無料/有料の機能差設定）
- **ペインポイント**:
  - 運用に手間をかけたくない（個人開発のため）
- **成功体験**:
  - 管理画面で必要な操作が素早くできる
  - 異常があれば即座に把握できる

#### U-03: ゲスト（閲覧のみ）[将来拡張]

- **役割**: 一般ユーザーが公開したカレンダーの空き時間を閲覧し、予約を行う
- **主な利用目的**: 友人・知人と予定を調整する
- **利用シーン**:
  - 一般ユーザーから共有されたリンクを開く
  - 空き時間を確認して予約リクエストを送る
- **利用デバイス**: PC / スマホ（ブラウザ）
- **ITリテラシー**: 不問（リンクを開くだけで使える必要がある）
- **主要タスク**:
  1. 公開カレンダーの空き時間を閲覧する
  2. 予約リクエストを送信する（名前・目的等の最低限の情報入力）
- **ペインポイント**:
  - 複雑な操作は嫌。すぐに空き時間がわかりたい
- **成功体験**:
  - リンクを開いたら空き時間が一目でわかり、ワンステップで予約できる
