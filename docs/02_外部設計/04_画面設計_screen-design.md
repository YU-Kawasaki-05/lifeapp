# 画面設計
## プロジェクト名: ARDORS（アーダース）

---

## 1. デザインシステム概要

### 1.1 UIフレームワーク

| 項目 | 採用技術 | 理由 |
|------|---------|------|
| スタイリング | Tailwind CSS 4.x | ユーティリティファースト・高速UI構築 |
| コンポーネント | shadcn/ui | コピペ型・アクセシビリティ対応・デザイン統一 |
| テーマ | shadcn/ui のデフォルトテーマ + カスタムカラー | 一貫性あるデザイン |
| アイコン | Lucide React | shadcn/ui 標準。MIT ライセンス |
| グラフ | Recharts | React 対応・軽量 |
| カレンダー | react-big-calendar または @fullcalendar/react | タイムライン表示用 |

### 1.2 カラーパレット

```css
/* src/app/globals.css — Tailwind CSS 4.x のテーマ設定 */
@theme {
  /* ブランドカラー */
  --color-brand-50: #f0f4ff;
  --color-brand-100: #dbeafe;
  --color-brand-500: #3b82f6;  /* primary */
  --color-brand-600: #2563eb;
  --color-brand-700: #1d4ed8;

  /* セマンティックカラー */
  --color-success: #22c55e;    /* 達成・完了 */
  --color-warning: #f59e0b;    /* 注意・Warm状態 */
  --color-danger: #ef4444;     /* エラー・期限超過 */
  --color-info: #3b82f6;       /* 情報・GCalイベント */

  /* PJ状態カラー */
  --color-status-active: #22c55e;   /* Active = 緑 */
  --color-status-warm: #f59e0b;     /* Warm = 黄 */
  --color-status-cold: #94a3b8;     /* Cold = グレー */
}
```

### 1.3 タイポグラフィ

| 用途 | クラス | サイズ |
|------|--------|--------|
| ページタイトル | `text-2xl font-bold` | 24px |
| セクションタイトル | `text-lg font-semibold` | 18px |
| 本文 | `text-sm` | 14px |
| キャプション | `text-xs text-muted-foreground` | 12px |
| コード | `font-mono text-sm` | 14px |

### 1.4 スペーシング

Tailwind のデフォルトスケールを使用（4px ベース）。

- カード内パディング: `p-4`（16px）
- セクション間マージン: `mb-6`（24px）
- コンパクト表示: `p-2`（8px）

### 4.4 SCR-20: ダッシュボード

**URL**: `/dashboard`  
**認証**: 必要  
**対応 FR**: FR-60, FR-13, FR-41（チェック）  

時間帯に応じてファーストビューが変化する適応型ダッシュボード。

**時間帯区分**:
- 朝モード（起床〜正午）: AIブリーフィング + エネルギーチェックイン + タイムライン + 習慣
- 日中モード（正午〜17時）: 現在ブロック + 次ブロック + 残りタイムライン
- 夜モード（17時〜就寝）: Done List + 習慣達成状況 + デイリークローズ誘導

**PC レイアウト（朝モード）**:
```

│ ストリーミングテキスト表示                            │

│ チェックイン │                                       │
│ 3項目        │ 09:00 ██ API設計（開発PJ）             │
│ 各1タップ    │ 10:30 ░░ 休憩                          │
│              │ ...                                   │
├──────────────┤                                       │
│ 今日の習慣   │                                       │
│ (HabitCheck) │                                       │
│ 1タップ✓    │                                       │
└──────────────┴──────────────────────────────────────┘
```

## 5. インタラクションパターン

### 5.1 フォーム送信フロー

```
1. ユーザーが入力
2. フロントエンド Zod バリデーション（即時フィードバック）
3. 送信ボタン押下 → disabled + ローディングスピナー
4. Server Action 実行
5a. 成功: Toast 通知（緑）+ データ更新（楽観的更新またはリフレッシュ）
5b. 失敗: Toast 通知（赤）+ エラーメッセージ表示
6. ボタン再度有効化
```

### 5.2 削除確認ダイアログ

```typescript
// 論理削除でも確認ダイアログを表示
```

### 5.3 ローディング状態

| 状態 | UI |
|------|-----|
| ページ初期ロード | `<Skeleton>` コンポーネントで各カードの形を維持 |
| Server Action 実行中 | ボタン disabled + `<Loader2 className="animate-spin">` |
| AI 応答生成中 | ストリーミングテキスト表示（文字が順次表示） |
| GCal同期中 | ボタン disabled + プログレス表示 |

### 5.4 エラー表示

| エラー種別 | UI |
|----------|-----|
| フォームバリデーション | フィールド下にインラインエラー（赤テキスト） |
| Server Action 失敗 | `<Toast>` 右下に表示（3秒で消える） |
| ページレベルエラー | `error.tsx` のエラー境界 |
| ネットワークエラー | Toast + 「再試行」ボタン |

## 6. 音声入力（VoiceInputButton）設計

```typescript
// src/shared/ui/voice-input-button.tsx
'use client'

import { useState, useRef } from 'react'
import { Mic, MicOff, Loader2 } from 'lucide-react'
import { Button } from '@/shared/ui/shadcn/button'

type VoiceInputState = 'idle' | 'listening' | 'processing'

interface VoiceInputButtonProps {
  onTranscript: (text: string) => void
  language?: string  // default: 'ja-JP'
}

export function VoiceInputButton({ onTranscript, language = 'ja-JP' }: VoiceInputButtonProps) {
  const [state, setState] = useState<VoiceInputState>('idle')
  const recognitionRef = useRef<SpeechRecognition | null>(null)

  const handleToggle = () => {
    if (state === 'listening') {
      recognitionRef.current?.stop()
      setState('idle')
      return
    }

    if (!('webkitSpeechRecognition' in window) && !('SpeechRecognition' in window)) {
      // Web Speech API 非対応ブラウザ（Safari iOS 等）
      alert('お使いのブラウザは音声入力に対応していません。Chrome または Edge をご利用ください。')
      return
    }

    const SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition
    const recognition = new SpeechRecognition()
    recognition.lang = language
    recognition.continuous = false
    recognition.interimResults = false

    recognition.onstart = () => setState('listening')
    recognition.onresult = (event) => {
      const transcript = event.results[0][0].transcript
      onTranscript(transcript)
      setState('idle')
    }
    recognition.onerror = () => setState('idle')
    recognition.onend = () => setState('idle')

    recognition.start()
    recognitionRef.current = recognition
  }

  return (
    <Button
      variant="ghost"
      size="icon"
      onClick={handleToggle}
      className={state === 'listening' ? 'text-red-500 animate-pulse' : ''}
      aria-label={state === 'listening' ? '録音停止' : '音声入力開始'}
    >
      {state === 'listening' ? <MicOff /> : <Mic />}
    </Button>
  )
}
```
