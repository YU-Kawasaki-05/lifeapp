# 用語集（Glossary）
## プロジェクト名: ARDORS（アーダース）

---

## プロダクト用語

| 用語 | 読み | 定義 |
|------|------|------|
| ARDORS | アーダース | プロジェクト名。AIを搭載した個人向け生産性・タスク・プロジェクト・習慣・内省の統合管理パートナーアプリ |
| ブレインダンプ | - | 音声またはテキストで思考を自由に書き出す入力方式。AIが内容を自動分類（タスク/ビジョン/PJ候補）して構造化する |
| タイムボクシング | - | 1週間分のスケジュールをAIが生成するプロセス。タスクを時間ブロックに割り当て、AIが承認を得てからGoogle Calendarにpushする |
| タイムブロック | - | タイムボクシングで生成される時間の区切り。開始時刻・終了時刻・内容・紐付くPJ/タスクを持つ |
| コールドスタート | - | 新規ユーザーがデータを持っていない初期状態。ARDORSではオンボーディングでデータを一括収集し解消する |
| モーニングブリーフィング | - | 朝にダッシュボードを開いた際にAIが1日1回行う「今日の準備」。今日のスケジュール・最重要タスク・昨日からの引き継ぎを含む |
| デイリークローズ | - | 1日の終わりに行う振り返りフロー。ユーザーがまとめて入力→AI分析→対話で深掘り |
| ウィークリーレビュー | - | 週1回の振り返り・方向修正・来週計画のフロー。AIコーチモードが発動する |
| AIコーチモード | - | ウィークリー/月次レビュー時にAIのトーンが切り替わり、データに基づく率直・建設的なフィードバックを行う状態 |
| 建設的AIの原則 | - | 全AI対話において「満足させる言葉だけでなく、目的達成に向けた客観的視点を提供する」設計原則 |
| 探索タイム | - | AIがスケジュール生成時に自動組み込む自由な思考・学習・休息のための時間枠。DMN（デフォルトモードネットワーク）活性化を目的とする |
| AI非依存の基盤保証 | - | LLM APIが障害・利用不能の場合でも、手動CRUDで全機能が動作することを保証する設計原則 |
| ゴール・ジャーニーマップ | - | 長期ゴール→中期目標→週次ゴール→タスクの階層ツリーをプログレスバー付きで可視化したもの |
| ヘルススコア | - | プロジェクトの健全度を自動算出した指標。最終作業日・タスク消化率・期限との距離・ブロッカーの有無から算出。赤/黄/緑で表示 |
| if-then plan | - | 「もしXなら、Yをする」の形式で習慣を定義する方式。Tiny Habitsの研究に基づく |
| cue（きっかけ） | キュー | 習慣のトリガー。「朝食後」「PCを開いたら」等の特定の状況を指定する |
| K/AL統合 | - | ナレッジキャプチャとActionable Listを1機能に統合した設計判断。AIが内容を自動判別して振り分ける |
| セレンディピティ・エンジン | - | ナレッジキャプチャに溜まったメモをAIが定期スキャンし、無関係に見えるメモ間の接点を提示するP2機能 |
| コンテキスト復元 | - | 数日間触っていなかったPJを再開する際に、AIが前回の作業内容・次のアクション・ブロッカーを提示する機能 |

## 設計原則（5原則）

| 原則名 | 内容 |
|--------|------|
| 入力コスト最小化 | 全入力はスキップ可能。最小1タップ。管理のめんどくさを排除する |
| AI非依存の基盤保証 | LLM API障害時でも手動CRUDで全機能が動作する。多層防衛 |
| 建設的AIの原則 | 全AI対話で目的達成に向けた客観的視点を提供。褒めるだけの対話は禁止 |
| コールドスタート設計 | 初日はAIなし基盤で価値提供。段階的にAIの精度が上がる |
| プラットフォーム戦略 | Webファースト（レスポンシブ）。モバイルネイティブは将来拡張 |
