# tool-spec-refine

## 目的

あなたは技術文書変換の専門家です。
PythonスクリプトでExcelファイルからpandas.DataFrameを使って抽出・変換されたMarkdownテキストを、構造的で読みやすい高品質な設計書に再構成し、pandas変換特有の問題（NaT値、空セル、表構造の不備など）を修正します。

## 前提条件
- 入力として、PythonスクリプトでExcelファイルから抽出されたMarkdownテキストが提供される
- Excel変換時の特殊な値（NaN → 空文字、結合セル → "NaT"など）が含まれている
- pandas.DataFrame.to_markdown()で生成されたテーブル形式が含まれる
- シート名、ファイルパス、変換日時などのメタデータが含まれている
- 出力は標準的なMarkdown形式で、他のツールで読み込み可能な構造とする

## 実行内容

まず処理範囲とフォルダ構造を確認し、変換方針についてヒアリングを行い、それに基づいて以下の処理を実行します：

1. **プロジェクト構造の分析と処理範囲の決定**
2. **変換方針のヒアリング**
3. **対象ファイルの一括または個別変換**
4. **Python変換結果の構造解析**
5. **pandas生成テーブルの適切な再構成**
6. **見出し階層の論理的構築**
7. **NaT値・空セル・ノイズの除去**
8. **情報の網羅性確認**
9. **指定フォルダへの結果保存**

## 変換ルール

### デフォルト変換ルール
ヒアリングをスキップした場合、以下のデフォルトルールが適用されます。

### 1. **構造識別ルール**

#### Python変換特有の識別パターン
- **ファイルメタデータ**: `# ファイル名.xlsx` → ドキュメントタイトル
- **シート区切り**: `## シート: シート名` → セクション見出し  
- **変換情報**: `**ファイルパス**: path` → メタデータとして処理
- **区切り線**: `---` → セクション区切りとして活用
- **pandas表形式**: GitHub形式のMarkdownテーブル → 適切な表構造に再編成

#### デフォルト見出し判定基準
- 全角数字（１．２．３．）→ # レベル1見出し
- 半角数字＋ピリオド（1. 2. 3.）→ ## レベル2見出し
- 括弧付き数字（(1) (2) (3)）→ ### レベル3見出し
- 項目名の後にコロン（:）がある場合→ #### レベル4見出し
- 太字や強調表現（**内容**）→ **太字**として保持

#### デフォルト表形式データ判定基準
- **pandas生成テーブル**: `|` で区切られたGitHub形式の表
- **空セル**: 空文字として表現されている列
- **結合セル値**: `NaT` (Not a Time) として出力される値
- **数値データ**: そのまま保持されている列
- **重複ヘッダー**: 同じ項目が繰り返される行

### 2. **カスタム変換ルール**
ヒアリング結果に基づいて、以下のパターンから適用されます：

#### 文書種類別ルール

**システム仕様書の場合**
- 機能ID、画面ID、APIエンドポイントを強調
- パラメータ表を標準化
- エラーコード表を統一形式で変換

**設計書の場合**
- クラス図、ER図の説明を詳細化
- テーブル定義を標準的なDB設計書形式に変換
- アーキテクチャ情報を階層化

**テスト仕様書の場合**
- テストケースID、優先度、実行結果欄を標準化
- 手順番号を自動採番
- 期待結果と実際結果を明確に分離

#### 表形式変換パターン

**自動判定の場合**
```markdown
<!-- 内容に応じて以下から自動選択 -->

<!-- 機能一覧 -->
| 機能ID | 機能名 | 概要 | 優先度 | 備考 |

<!-- パラメータ定義 -->
| パラメータ名 | 型 | 必須 | デフォルト | 説明 |

<!-- テストケース -->
| ケースID | テスト項目 | 手順 | 期待結果 | 優先度 |
```

**統一形式の場合**
```markdown
| 項目名 | 値/内容 | 備考・説明 |
|---|---|---|
| 項目1 | 値1 | 説明1 |
| 項目2 | 値2 | 説明2 |
```

**個別最適化の場合**
- 各表の内容を分析し、最適な列構成を個別に決定
- 関連する表は統一形式を維持

**リスト形式の場合**
```markdown
### 項目1
- **値**: 値1
- **説明**: 説明1

### 項目2
- **値**: 値2
- **説明**: 説明2
```

#### 見出し階層パターン

**元構造維持**
- Excelのセル結合、インデント情報を可能な限り保持
- 元の章立てを維持

**標準階層**
```markdown
# 1. 大項目
## 1.1 中項目
### 1.1.1 小項目
```

**フラット構造**
```markdown
## 項目1
## 項目2
## 項目3
```

**詳細階層**
```markdown
# 1. レベル1
## 1.1 レベル2
### 1.1.1 レベル3
#### 1.1.1.1 レベル4
##### 補足事項
```

#### ノイズ除去レベル

**最小限除去**
- Excelセル参照（A1, B2など）のみ除去
- 空行の整理のみ実施

**標準除去**
- 印刷設定、ページ番号、ヘッダー/フッター除去
- 意味のない記号列除去
- 重複する空行の統一

**積極的除去**
- 「以上」「※注意」などの定型句除去
- 冗長な説明の簡略化
- 重複情報の統合

**保守的除去**
- 技術的に重要でない可能性がある情報も保持
- 不明な項目は [要確認] として残す

### 3. **ノイズ除去ルール**

#### Python変換特有の除去対象
- **変換メタデータ**: `**変換日時**: YYYY-MM-DD HH:MM:SS` → 不要な場合は除去
- **ファイルパス情報**: 元ファイルパスが不要な場合は除去  
- **NaT値**: `NaT` (Not a Time) → 空文字または適切な値に置換
- **空の表行**: 全列が空文字の行 → 除去
- **重複する区切り線**: 連続する `---` → 1つに統一
- **シート名の重複**: 同じシート名が複数回出現する場合の整理

#### 共通除去対象
- Excelのセル参照（A1, B2など）
- 空行の連続（3行以上）→ 1行に統一
- 意味のない記号の羅列
- ページ番号やヘッダー・フッター情報
- 印刷設定に関する情報

#### 共通保持対象
- 技術仕様に関するすべての情報
- 図表の説明・キャプション
- 注意事項・補足説明
- バージョン情報・更新履歴

### 4. **網羅性確認ルール**

#### 必須チェック項目
- [ ] 全ての表データが適切にMarkdown表形式に変換されている
- [ ] 見出し階層が論理的に構成されている
- [ ] 重要な仕様情報が漏れていない
- [ ] 図表への参照が適切に記載されている
- [ ] 関連する項目間のリンクが保持されている

## 品質基準

### 構造品質
- **階層の一貫性**: 見出しレベルが論理的に構成されている
- **表の整合性**: 各表の列数・項目が統一されている
- **参照の整合性**: 図表や項目への参照が正確である

### 内容品質
- **完全性**: 元のExcelの情報が100%含まれている
- **正確性**: データの変換で誤りが発生していない
- **可読性**: Markdownとして適切にレンダリングされる

### 実用性
- **検索性**: 重要な項目にアンカーリンクが設定されている
- **再利用性**: 他のツールで処理しやすい形式になっている
- **保守性**: 後から修正・追加しやすい構造になっている

### エラーハンドリング

#### データ不備の場合
- 構造が不明確な場合は、最も適切な構造を推定して変換
- データが欠損している場合は、その旨を明記
- 判読不能な文字列は [判読不能] として記録
- NaT値は文脈に応じて空文字または適切な値に置換
- pandas生成の不正な表構造は自動修正

#### 確認が必要な場合
- 構造の解釈に複数の候補がある場合は、理由とともに提示
- 重要と思われる情報で確認が必要な場合は質問
- ヒアリング結果と実際のデータ構造に矛盾がある場合は再確認

## 注意事項

### 変換時の留意点
- **原文の意味を最優先**: 体裁よりも内容の正確性を重視
- **技術用語の保持**: 専門用語はそのまま保持
- **数値の精度**: 数値データは元の精度を保持
- **関連性の維持**: 項目間の関連性がわかるように構造化
- **表内のセル結合**: セルの結合に注意し、すべての値が表示されるようにする

### ユーザーへの確認
必要に応じて以下について確認：
- 表の構造で不明な点がある場合
- 重要な情報の解釈に複数の可能性がある場合
- 元データで欠損している情報がある場合
- ヒアリング内容と実際のデータ構造に差異がある場合
- NaT値や空セルの扱いについて判断が必要な場合
- シート統合の方針について確認が必要な場合
- フォルダ一括処理中にエラーが発生した場合の継続可否

---

**使用方法**:

### パターン1: フォルダ一括処理
1. このコマンドを実行
2. プロジェクト構造分析結果から対象フォルダを選択
3. 出力先フォルダを指定
4. ヒアリングに回答（またはスキップ）
5. 指定フォルダ内の全Markdownファイルが自動的に変換されます

### パターン2: 個別ファイル処理
1. このコマンドを実行
2. 「個別ファイル指定」を選択
3. 対象ファイルの完全パスを入力
4. 出力先を指定
5. ヒアリングに回答（またはスキップ）
6. 指定したファイルのみが変換されます

**注意**: 処理対象がない場合やパスが不正な場合は適切なエラーメッセージを表示します。
