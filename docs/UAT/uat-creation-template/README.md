# UAT仕様書自動作成ツール - 実行ガイド

## 概要

このツールは、Claude Codeを使用してUAT（ユーザー受入テスト）仕様書を自動生成します。
設計書から業務カテゴリ別にUAT仕様書を作成し、検証・改善を行います。

## 前提条件

- Claude Code CLI がインストールされていること
- ターミナルから `claude` コマンドが実行可能であること

Claude Code のインストール方法: https://www.claudecode.com/

## ディレクトリ構成

```
uat-creation-template/
├── .claude/                      # Claude Code設定ファイル群
│   ├── agents/                   # サブエージェント定義
│   │   ├── file-analyzer.md      # ファイル解析エージェント
│   │   ├── uat0-make-scope-agent.md   # スコープ定義書作成
│   │   ├── uat1-make-temp-agent.md    # 仮UAT仕様書作成
│   │   ├── uat2-validate-agent.md     # UAT仕様書検証
│   │   └── uat3-make-uat-agent.md     # UAT仕様書改善
│   ├── commands/                 # カスタムコマンド定義
│   │   ├── input-prepare.md      # 入力準備コマンド
│   │   └── make-uat-complete.md  # UAT作成メインコマンド
│   ├── settings.local.json       # 権限設定
│   └── uat-config.json           # プロジェクト設定（カスタマイズ可）
├── 01-inputs/                    # 設計書を配置（Markdown形式）
│   └── (ここに設計書を配置してください)
├── AI-generated/                 # 自動生成される成果物
│   └── (実行後に自動作成されます)
└── README.md                     # 本ファイル
```

## セットアップ手順

### 1. 作業ディレクトリの準備

このテンプレートディレクトリ全体を、UAT作成を行いたいプロジェクトの作業場所にコピーしてください。

```bash
# 例: my-uat-projectという名前でコピー
cp -r uat-creation-template my-uat-project
cd my-uat-project
```

### 2. プロジェクト設定のカスタマイズ（オプション）

`.claude/uat-config.json` をプロジェクトに合わせて編集してください。

```json
{
    "project": {
        "name": "プロジェクト名"  // ← プロジェクト名を変更
    },
    "inputPaths": {
        "rootDir": "./01-inputs",
        "businessListPattern": "業務一覧表.md"  // ← 業務一覧ファイル名
    },
    "businessStatus": {
        "targetStatuses": [
            "🔍 システムレビュー待ち",  // ← 対象とする業務ステータス
            "👀 システムレビュー中",
            "📋 業務側レビュー待ち",
            "👥 業務側レビュー中",
            "✅ 承認済み"
        ],
        "excludeStatuses": [
            "❌ 未作成",  // ← 除外する業務ステータス
            "📝 未着手"
        ]
    },
    "coverageTarget": 100  // ← 目標業務網羅率(%)
}
```

### 3. 設計書の配置

`01-inputs/` ディレクトリに、以下の形式でMarkdown形式の設計書を配置してください。

**必須ファイル:**
- `業務一覧表.md` - 全業務の一覧（業務ID、業務名、ステータスを含む）

**その他の設計書:**
- 業務フロー図
- 画面設計書
- データ設計書
- API仕様書
- など、UAT作成に必要な設計書

**ファイル名の推奨形式:**
- `BF-XXX_業務名.md` (業務フロー)
- `画面設計_XXX.md` (画面設計)
- など、業務IDを含むファイル名にすると自動的に関連付けされます

### 4. Claude Code設定の確認（オプション）

Claude Codeの設定を最適化する場合、以下を実行してください。

```bash
claude
```

起動後、以下を入力:

```
/config
```

以下の設定を行います:
- **Auto-compact**: `false` に設定（Context量を増やすため）

## 実行方法

### 基本的な実行

1. ターミナルで作業ディレクトリに移動

```bash
cd my-uat-project
```

2. Claude Codeを起動

```bash
claude
```

3. UAT作成コマンドを実行

Claude Codeのプロンプトで以下を入力:

```
/make-uat-complete
```

または、改善回数を指定する場合:

```
/make-uat-complete 3
```

- 引数: 改善回数（省略時は1回）
- 例: `/make-uat-complete 5` で5回改善を繰り返します

### 入力準備コマンド（オプション）

プロジェクトの `docs/` ディレクトリから必要な設計書のみを選別できます。

#### ソースとターゲット

- **ソース**: プロジェクトルートの `docs/` ディレクトリ（テンプレートからの相対パス `../../`）
- **除外**: `docs/UAT/` は分析対象から除外（循環参照防止）
- **ターゲット**: `01-inputs/` に選別された設計書がコピーされる

#### 実行方法

Claude Codeで以下を実行:

```
/input-prepare
```

自動的に `docs/` ディレクトリ内のファイルを分析し、UAT作成に必要なファイルを `01-inputs/` にコピーします。

#### 機能

このコマンドは以下を自動的に行います:

- `docs/` ディレクトリ内の全Markdownファイルを分析（`docs/UAT/` は除外）
- file-analyzerサブエージェントによるUAT観点での有用性評価
- 必要なファイルのみを `01-inputs/` にコピー
- 不要なファイルを除外（処理時間とトークン削減）

#### 選別基準

- **High Priority**: ユーザー操作、画面仕様、業務フロー、API設計
- **Medium Priority**: システム概要、データ仕様、セキュリティ
- **Low Priority**: 非機能要件、技術仕様
- **除外**: 開発環境設定、プロジェクト管理、テスト仕様

#### 利点

- **処理時間短縮**: 不要な設計書を読み込まない
- **トークン削減**: 関連する設計書のみを処理
- **精度向上**: ノイズとなる情報を削減

#### 手動選別する場合

`/input-prepare` を使わず、手動で `01-inputs/` に設計書を配置することも可能です。

## 成果物

実行完了後、`AI-generated/` ディレクトリに以下のファイルが生成されます。

### 主要成果物

```
AI-generated/
├── UATスコープ定義書.md                    # 全対象機能のスコープ定義
├── uat_specs/                              # カテゴリ別UAT仕様書
│   ├── UAT仕様書_A1_チャット.md
│   ├── UAT仕様書_A2_画像添付.md
│   ├── UAT仕様書_B1_認証.md
│   └── ... (全カテゴリ分)
├── 仮UAT仕様書_統合サマリー.md              # 仮仕様書の統合情報
├── uat_reviews/                            # カテゴリ別検証結果
│   ├── UAT仕様書_A1_チャット_検証結果.md
│   └── ... (全カテゴリ分)
├── UAT仕様書_統合検証レポート.md            # 統合検証レポート
└── UAT仕様書_統合レポート.md                # 最終統合レポート
```

### バックアップ

改善前のUAT仕様書は自動的にバックアップされます:

```
AI-generated/uat_specs/backup/
└── UAT仕様書_A1_チャット_20260310_120000.md
```

## 処理フロー

1. **フェーズ1: スコープ定義**
   - 業務一覧表から対象業務を抽出
   - 業務カテゴリ構造を含むスコープ定義書を作成

2. **フェーズ2: 仮UAT仕様書作成**
   - 各業務カテゴリごとに仮UAT仕様書を生成
   - 全カテゴリを自動的に処理

3. **フェーズ3: 検証**
   - 各カテゴリのUAT仕様書を個別に検証
   - 品質・網羅性・実行可能性をチェック

4. **フェーズ4: 改善**
   - 検証結果に基づき各カテゴリを改善
   - 指定回数だけ改善を繰り返す

## トラブルシューティング

### Claude Codeが起動しない

```bash
# Claude Codeのバージョン確認
claude --version

# 再インストールが必要な場合
# https://www.claudecode.com/ から最新版をインストール
```

### 権限エラーが発生する

`.claude/settings.local.json` の `permissions.allow` に必要な権限が含まれているか確認してください。

### 処理が途中で止まる

1. Claude Codeを再起動
2. `/make-uat-complete` を再実行（途中から再開されます）

### 業務が正しく抽出されない

1. `01-inputs/業務一覧表.md` のフォーマットを確認
2. `.claude/uat-config.json` の `businessStatus` 設定を確認
3. `/input-prepare` コマンドで対象業務を確認

## 注意事項

- **実行時間**: プロジェクト規模により30分〜2時間程度かかります
- **トークン使用量**: 改善回数が多いほどトークン使用量が増加します
- **ファイル配置**: 設計書は必ず `01-inputs/` に配置してください
- **Markdown形式**: 設計書は必ずMarkdown形式(.md)で用意してください

## カスタマイズ

### テンプレートの変更

各エージェントの動作をカスタマイズする場合、`.claude/agents/` 内のファイルを編集してください。

### コマンドの追加

新しいコマンドを追加する場合、`.claude/commands/` にMarkdown形式で定義を追加してください。

## サポート

問題が発生した場合は、以下を確認してください:

1. Claude Codeのバージョンが最新か
2. 設計書のMarkdown形式が正しいか
3. `.claude/uat-config.json` の設定が正しいか
4. `01-inputs/` に必要なファイルが配置されているか
